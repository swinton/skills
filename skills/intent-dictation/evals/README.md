# Evaluation cases

These cases are behavioral examples, not a full benchmark. Expected outputs are reference answers, and other wording can be valid.

The invariants matter more than exact phrasing. They identify meaning, uncertainty, technical text, and responsibility that must survive the transformation.

When a real-world result fails, add a fictionalized, non-sensitive version here before changing the instructions. CI invokes each self-contained prompt and uses a separate structured judge call to evaluate the invariants; it does not exact-match the reference output.
